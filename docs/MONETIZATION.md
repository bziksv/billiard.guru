# Платные функции (задел)

Документ описывает запланированные платные возможности для клубов на billiard.guru. Реализация подписки и биллинга — отдельный этап.

## 1. Рассылка новости игрокам города (первая кандидатура)

**Суть:** после модерации и публикации на сайте клуб может дополнительно разослать новость подтверждённым игрокам **города клуба** в Telegram (и при необходимости другие мессенджеры).

**Сейчас в продукте:**

| Место | Поведение |
|-------|-----------|
| `/manage/clubs/[id]/news` | Галочка «Разослать игрокам города в мессенджеры» — **disabled**, подпись о платной подписке |
| `ClubNews.cityBroadcastRequested` | Поле в БД, по умолчанию `false` |
| `/admin/club-news` | Колонка «Рассылка»: «—» или «Рассылка по городу» |
| API POST `/api/clubs/[id]/news` | Флаг с клиента пока **не активируется** (всегда `false`) |
| Telegram модерации | При `cityBroadcastRequested === true` — строка в сообщении админу |

**Не путать с:**

- уведомлением **суперадмину** о модерации (бесплатно, всегда);
- уведомлением **клубу** об одобрении/отклонении (бесплатно).

**При включении подписки (TODO):**

1. Проверка подписки клуба перед приёмом `cityBroadcastRequested: true`.
2. Активная галочка в форме новости для клубов с подпиской.
3. После `approve` — рассылка через каталог уведомлений (opt-in, лимиты по тарифу).
4. Аудит: `club.news.city_broadcast`.

**Связанные файлы:** `club-news-display.ts`, `club-news-moderation.ts`, `club-news-panel.tsx`, `club-news-table.tsx`, `schema.prisma` (`ClubNews.cityBroadcastRequested`).
