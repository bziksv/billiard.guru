## UI карточки (эталон)

1. **Шапка:** только `#{номер}` — без «Тур N» и «сл.N».
2. **Две строки игроков** + при необходимости строка **форы**.
3. **Подвал** (до 3 строк, `gridFooterHeight(3)`):
   - диапазон мест (для #13–#27) — по **номеру на карточке** (`matchNumber`), не по ошибочному `round/slot` + legacy `maxRound`;
   - `проигравший на #…` / `× на #…` (bye);
   - `победитель на #…` / `автопроход на #…`.
4. **Заголовки колонок** — названия **этапов** («1/4 финала», «Полуфинал»), не «место 5–8».
5. Высота карточки: `FIXED_SWISS_CARD_H`; вертикальный шаг: `FIXED_SWISS_BRACKET_UNIT` (184px).

Логика: `fixedSwissTs27PlacementByMatchNo()` / `fixedSwissPlacementLabel(..., matchNumber)`.

## Линии SVG (кратко)

Полная таблица — `docs/FIXED_SWISS_BRACKET_LINES.md`.

- **Рисуем:** вилки R1 (win/loss); короткие win R2→R3 (#9→#21 … #16→#20); вилки 1/4→полуфинал и полуфинал→финал.
- **Не рисуем:** переходы loss в крест/1/4, крест→1/4 — только текст в подвале.
- Точка привязки линии: `teamDividerY`, не центр слота.

## Модалка результата (эталон)

- **Сейчас** (начало) — ставит время и **сразу сохраняет** в БД.
- **Сейчас** (конец) — конец (+ начало, если пусто) и сохранение.
- **Зафиксировать результат** — при пустых полях подставляет «сейчас»; не отправляет `null`, чтобы не затереть уже сохранённое время.
- На сервере при первой фиксации победителя — автозаполнение `startedAt` / `finishedAt`, если пусто.

## Нумерация встреч (#)

- Отображаемый # = порядок после сортировки по `fixedSwissMatchNo()` (`buildMatchNumbers` в layout).
- Для подписей мест при 27 встречах приоритет у **`matchNumber`** в `fixedSwissPlacementLabel`.

## Как строить другую сетку «как эталон»

1. **Шаблон:** `FixedSwissTemplate` — явный список `matches` + `links[]` (win/loss, round, slot).
2. **Layout:** отдельная функция позиций (как `buildTsPositions27`) — Y по слотам, без «плавающих» mid между несвязанными парами.
3. **Колонки:** отдельная `*MatchCol(round, slot)` под число колонок формата.
4. **Линии:** явный whitelist в `shouldDraw*WinEdge` / `shouldDraw*LossEdge`; не рисовать «всё подряд» fork’ом между соседними колонками.
5. **UI:** переиспользовать `LlbBracketMatch` + `MatchResultModal`; вынести отличия в layout/labels, не дублировать карточку.
6. **Тест:** скрипт по образцу `test-fixed-swiss-layout.ts` (links, Y, #, линии, коллизии колонок).
7. После смены links в коде — **«Сформировать заново»** в админке.
