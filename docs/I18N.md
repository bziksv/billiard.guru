# Международная версия billiard.guru (i18n)

Стратегия зафиксирована в `.cursor/rules/i18n-strategy.mdc`. **Сначала — английский до идеала**, затем zh и языки EU по спросу.

---

## 1. Принятые решения

| Тема | Решение |
|------|---------|
| Default locale | `ru`, URL **без** `/ru/` |
| English | `/en/…` |
| Библиотека | [next-intl](https://next-intl.dev/) + App Router |
| Определение языка | Cookie → URL → Accept-Language → **не** IP для языка |
| IP / geo | Только подсказка **региона** (фильтр страна/город), не редирект языка |
| Контент на `/en/` | **Только английский**; русский fallback — отдельная заглушка, не «как есть» |
| Admin / manage / cabinet | Вне scope до отдельной фазы |
| SEO | `hreflang`, отдельные title/description/keywords на EN |

---

## 2. Как это выглядит для пользователя

```
https://billiard.guru/tournaments          → русский UI + контент
https://billiard.guru/en/tournaments       → English UI + English content
https://billiard.guru/en/rules/pool        → правила на английском
```

**Переключатель в шапке:** EN | Русский | (позже 中文). Выбор → cookie `NEXT_LOCALE` на 1 год.

**Первый визит** на `/` без cookie: если `Accept-Language` предпочитает `en` — мягкий редирект на `/en/` **или** баннер «Continue in English» (решение на этапе 1.3).

**Тот же slug** на другом языке: `/rules/pyramid` ↔ `/en/rules/pyramid`.

---

## 3. Архитектура (целевая)

```
apps/web/
  messages/
    ru.json          # UI-строки (default)
    en.json
  src/
    i18n/
      routing.ts     # locales, defaultLocale, prefix
      request.ts     # next-intl server config
    middleware.ts    # locale detection, cookie, redirect
    app/
      (site)/        # ru — без [locale], legacy URLs
        layout.tsx
        page.tsx
        tournaments/...
      [locale]/
        (site)/      # en, позже zh, de, …
          layout.tsx
          page.tsx
          ...
  src/lib/billiard-rules/
    content/         # общие типы
    ru/              # pyramid-content.ts, … (миграция)
    en/              # pyramid-content.en.ts, …
```

**Контент правил:** одна структура (`BilliardTableType`, `BilliardHistory`), тексты по locale в `ru/` и `en/`.

**База (позже):** `City.nameEn`, локализованные поля клубов — фаза 5.

---

## 4. Приоритет языка (middleware)

```
1. NEXT_LOCALE cookie (explicit user choice)
2. Path prefix /en/, /zh/, …
3. Accept-Language (first visit only)
4. defaultLocale → ru
```

Geo IP **не участвует** в шагах 1–3.

---

## 5. Правило публикации контента

> **Если информация публикуется на сайте и раздел уже в scope EN — она должна быть на английском на `/en/…` в том же PR или до merge.**

| Ситуация | Действие |
|----------|----------|
| Новая публичная страница в scope EN | RU + EN тексты, EN metadata, hreflang |
| Правка `/rules/*` | Обновить RU **и** EN файл (или заглушка + noindex EN до перевода) |
| Новый блок истории / SEO | EN параллельно RU |
| Только RU готов | EN URL не в sitemap; переключатель не ведёт на `/en/…` этой страницы |
| Данные БД (имена, города) | Оригинал как есть; UI-подписи на EN («City», «Country») |

**Антипаттерн (запрещён):** пользователь на `/en/rules/pool` видит русские абзацы «как временно».

## 7. Фаза 0 — Инфраструктура

### Цель
Работающий `/en/` с переключателем и cookie; без массового перевода контента.

### Чеклист

- [ ] Установить `next-intl`
- [ ] `src/i18n/routing.ts`: `locales: ['ru', 'en']`, `defaultLocale: 'ru'`, `localePrefix: { ru: 'never', en: 'always' }` (или эквивалент)
- [ ] `src/middleware.ts`: cookie, Accept-Language, rewrite/redirect
- [ ] Cookie `NEXT_LOCALE`, SameSite=Lax, max-age 1 year
- [ ] Структура `app/[locale]/(site)/` для `en`; текущий `(site)` = ru
- [ ] `messages/ru.json`, `messages/en.json` — перенос `SITE_COPY` из `site.ts`
- [ ] Хелпер `t('nav.tournaments')` вместо прямого `SITE_COPY`
- [ ] Компонент `LocaleSwitcher` в шапке
- [ ] Переключение сохраняет path: `/tournaments` → `/en/tournaments`
- [ ] **Не** редиректить язык по IP
- [ ] Документировать в PR: geo только для фильтров

### Критерий готовности
- [ ] `curl -I http://localhost:3010/en/` → 200
- [ ] Выбор EN в switcher → cookie → следующий визит на `/` редиректит или открывает `/en/` по политике 1.3
- [ ] Русские URL без изменений: `/tournaments` → 200

## 8. Фаза 1 — English UI shell

### Цель
Публичная оболочка полностью на английском на `/en/*`.

### Scope страниц (оболочка)
- [ ] Layout: header, footer, nav
- [ ] `/en/` home
- [ ] `/en/login` (форма + labels; можно noindex)
- [ ] Пустые состояния, кнопки, фильтры geo (labels)
- [ ] Ошибки 404 / not-found для locale
- [ ] Fallback: `EnNotAvailable` компонент для страниц без перевода

### Чеклист перевода UI
- [ ] Все ключи из `SITE_COPY` → `en.json`
- [ ] Компоненты с hardcoded RU в `(site)/` — grep `[А-Яа-я]` в `components/site/`
- [ ] Формы: login, поиск, фильтры
- [ ] Даты/числа: `en-US` формат на `/en/`
- [ ] `aria-label`, placeholder, toast messages

### Критерий готовности
- [ ] На `/en/` нет русских строк в header/footer/hero labels
- [ ] Страницы **без** EN-контента показывают заглушку, не RU body

## 9. Фаза 2 — Lists & detail pages (EN)

### Цель
Списки и карточки: UI на EN; пользовательский **описательный** текст на EN где есть.

### Приоритет
1. [ ] `/en/tournaments`, `/en/tournaments/[id]`
2. [ ] `/en/clubs`, `/en/clubs/[id]`
3. [ ] `/en/players`, `/en/players/[id]`
4. [ ] `/en/coaches`, `/en/coaches/[id]`
5. [ ] `/en/pokatat`, `/en/ideas`

### Чеклист
- [ ] Статические подписи секций (Participants, Bracket, …) → `en.json`
- [ ] SEO: `generateMetadata` с locale `en`
- [ ] Имена собственные / названия турниров — **не переводить**
- [ ] Описания турниров/клубов: если только RU в БД — на EN показать notice «Description available in Russian» + link, **не** сырой RU параграф (до фазы 5)
- [ ] Фильтр country/city — EN labels; названия городов из БД пока как есть

### Критерий готовности
- [ ] Каждый URL из scope открывается на `/en/` без русских UI-строк
- [ ] Нет русских **абзацев** описаний на EN без явной пометки

## 15. Компонент «нет перевода»

Пока EN не готов:

- Заголовок: **This page is not available in English yet**
- Кнопка: **Read in Russian** → RU URL
- `robots: noindex` на EN заглушке
- Не показывать пункт в EN nav или помечать «Soon»

---

## 16. Workflow для агента / разработчика

1. Перед PR с публичным контентом — проверить scope фазы в этом документе.
2. Если раздел в scope EN — добавить/обновить EN файлы и `en.json` ключи.
3. Не мержить EN URL с русским body.
4. После изменений: `curl /en/…` + grep кириллицы (см. rule `i18n-strategy.mdc`).
5. zh/EU — **не начинать**, пока в этом файле фазы 0–4 не отмечены выполненными (чеклисты выше).
